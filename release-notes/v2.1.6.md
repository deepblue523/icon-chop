# IconChop 2.1.6

**Release date:** 2026-04-24  

This release improves everyday workflows (recent files, naming presets, and image generation), refreshes toolbar imagery, and expands local/GitHub release automation.

> **Version note:** When you publish, set the shipped version by bumping `<Version>` in `IconChop.csproj` and pushing tag `v<major>.<minor>.<patch>` (for example `v2.1.6`). The **Release** workflow builds with the version taken from the tag.

---

## Highlights

### Source images and Auto-name

- **Recent source images** — Up to 20 recently opened sheets are remembered. Use the new **history** control next to the source image actions to reopen a file from a thumbnail menu.
- **Auto-name app context MRU** — Recent app descriptions used for Auto-name are stored (newest first, capped at 10) so you can reuse them quickly.
- **Auto-name description presets** — Define named presets (short label + full description for the API) in a dedicated editor. The naming dropdown orders presets with a **recent presets** list (up to 24). Your last chosen preset is restored on startup when it still exists.
- **Image generation + Auto-name** — New option to **append the Auto-name app description** to the image-generation prompt when you want generated sheets aligned with the same product context.

### Image → Generate…

- **Prompt MRU** — Recent generation prompts are saved (up to 10) and surfaced in the prompt UI alongside templates.
- **Reference image history** — Each reference slot can keep a rolling history (up to 20 snapshots) with **history** menus and thumbnails to swap references without reloading from disk every time.
- **Toolbar polish** — Several actions use updated or new 32×32 toolbar images for clearer affordances.

### Settings and UI

- **Output folder MRU** — Recent output directories are tracked (existing cap: 12) for faster folder selection.
- **Status feedback** — Clearer status updates on the main form during longer operations.

## Packaging and automation

- **`Create-Release-All.ps1`** — One entry point to build the framework-dependent zip, self-contained zip, and installer artifacts for the version read from the project (or after a `-Bump` patch/minor/major).
- **`Publish-Release.ps1`** — Build locally, then push the branch and `v<version>` tag so GitHub Actions can attach release assets. Supports `-Bump`, `-DryRun`, and `-ForceTag`.
- **WiX / installer** — Self-contained publish output is still harvested into **`IconChop.Setup.msi`**, with more reliable discovery of the built MSI under **`bin\x64\Release`** (and fallback paths). A **WiX Burn bundle** project under `bundle/` wraps the MSI for a single bootstrapper-style **Setup.exe** once that project is invoked from your build or release pipeline (the tagged **Release** workflow currently publishes the MSI and both zips).
- **`Publish-GitHubRelease.ps1`** — Adjustments for the current release asset layout.

## Upgrade notes

- Settings continue to live under `%USERPROFILE%\icon-chop\config.json`. New lists (`InputImageMru`, `AutoNameAppContextMru`, `AutoNameAppDescriptionPresets`, `AutoNamePresetMru`, `PromptMru`, etc.) are populated as you use the app; older configs load with empty lists for those fields.
